# Oplusシフト管理のGWS置き換え 企画メモ

> 作成: 2026-07-01 / ステータス: 企画（未着手）
> ゴール: 実装の道筋を固め、実行できそうな段階で上長にぶん投げる

## 1. 目的・効果
- 目的: アルバイトのシフト管理に使っているOplusを、GWS(Google Workspace)の
  機能で置き換える
- 効果: Oplus契約費のまるごと削減（GWSは既存契約内で完結）
- 前提: シフトの「登録・閲覧」のみが対象。勤怠/給与連携は別アプリ担当で範囲外
  → GWSが苦手な領域を最初から回避できる = 実現性が高い

## 2. 現状の運用
- 登録: アルバイト本人がOplusに直接入力（管理者がシフトを組むのではなく、
  各コマの定員=座席上限を見て、空き枠に本人が"勝手に入る"セルフエントリー型）
- 定員管理: 個々人の判断ベース（厳密なロックは不要）
- 閲覧: Oplusアプリを各自が確認
- Oplusの提供価値の本体 = 「シフトのぱっと見の分かりやすさ」
  → 置き換えでここを落とさないことが成否の鍵

## 3. 代替設計（GWS構成）
- バックエンド: Googleスプレッドシート（日付×コマのマトリクス、残枠は数式）
- 登録/変更: GAS Web App 経由
    - デプロイ設定 = 実行ユーザー:自分 / アクセス:全員(Anyone)
    - アルバイトはログイン不要でURLを開き、専用UIから登録
    - 裏でオーナー権限でスプシに書き込む（スプシ自体は共有しない）
- 閲覧: スプシをリンク共有(閲覧のみ) or Web App上に一覧表示
- スプシ本体は直接触らせない（バックエンド化）

```
アルバイトの私物スマホ
  ├─ 登録/変更 ─▶ GAS Web App (Anyone/実行:自分) ─▶ スプシに書き込み
  └─ 閲覧    ─▶ スプシをリンク共有(閲覧のみ) or Web App上に一覧表示
                   ※スプシ本体は直接触らせない = バックエンド
```

## 4. この構成の強み
- アカウント問題の解消: Web App も閲覧スプシもログイン不要
  → アルバイトの私物端末から会社GWSアカウントにログインさせずに済む
    (情シス観点の最大リスクを回避)
- UI/UX担保: Web Appのカスタム画面でOplus的な操作感を再現（スマホ最適化）
- データ保護: 書き込みはWeb App経由のみ = 定員チェック等を挟める、直接改ざん不可

## 5. リスク・要確認
- 【最優先】GWS組織設定で「全員(Anyone)公開」が許可されているか
  → NGだと構成の土台が崩れる。情シスに先に確認
- 匿名運用: 誰が登録したか特定不可 → 本人識別方法(名前プルダウン/簡易PIN)を設計で担保
- 同時登録の競合 → GAS LockService で排他制御
- アルバイト目線: 見やすさ/使い勝手がOplus比で落ちないUI設計が成否の鍵
- GAS Web Appの実行クォータ・レスポンス（規模次第、シフト用途なら通常問題なし）

## 6. 段階計画
- Phase 0: ①「全員公開」可否を情シス確認 ②Oplus契約費・規模の数字を把握
- Phase 1: 最小プロトタイプ(スプシ + Web App登録UI)を組んでスマホで実地検証
- Phase 2: UI/UX作り込み、本人識別・定員バリデーション実装
- Phase 3: 試験運用(一部アルバイト) → 本格移行 → Oplus解約

## 7. 実装体制（ちかげの実装工数はメインで取れない前提）
- 案A: たたき台を生成AI(Claude等)で作成し、本人は仕上げ・検証に回る
- 案B: 企画確定後、実装はアルバイトに委譲する
