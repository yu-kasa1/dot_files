# Notion週間レポート生成 (notion-weekly-report)

## 概要
Notionの「日報自分用」ページ配下にある日次トグルを集約し、
会社提出用の週間レポートを生成する。
日報内の「AI」トグル配下は成功談/失敗談問わず **独立した「AI活用トピック」セクション** に分離する。
生成後、同ページ先頭に `N月M週目` トグルとして書き戻す。

## 呼び出しタイミング
- 週報作成タイミング（通常は金曜）
- ユーザーが「週報まとめて」「Notionから週報作って」等と依頼した時

## 実行手順

### 1. 対象期間の決定
- **終了日**: 直近の金曜日（当日が金曜なら当日）
- **開始日**: 終了日の4日前（月曜日）
- ユーザーが期間を明示指定した場合はそれに従う
- 日報に該当日付のエントリが足りなくても、存在するぶんだけで生成する（ユーザーに欠損を明示）

### 2. Notionから日報取得
1. `mcp__claude_ai_Notion__notion-search` で `query: "日報自分用"` を実行
2. ヒットしたページのIDを `mcp__claude_ai_Notion__notion-fetch` に渡して全文取得（ページID例: `33c97be1-9463-8070-8968-d24ef6612d4e`）
3. `<details><summary>M/D</summary>` 形式のトグルを日付ごとにパース

### 3. 日報内容の分類
各日のトグルから以下を抽出:
- **やったこと** → 今週の主な取り組み
- **やっちまったこと** → 反省点・課題
- **できたこと** → 成果・ハイライト
- **AIトグル配下（成功談/失敗談）** → **AI活用トピック**（独立セクション）
- 「明日やること」は使わない

### 4. レポート整形
以下のフォーマットで出力する:

```markdown
### 週間レポート（M/D〜M/D）

**今週の主な取り組み**
- （やったことを重複マージして箇条書き）

**成果・ハイライト**
- （できたことを箇条書き）

**反省点・課題**
- （やっちまったことを箇条書き）

**AI活用トピック**
- ✅ （AI成功談）
- ⚠️ （AI失敗談）
```

### 5. ユーザー確認
- 生成したレポートを提示し、修正希望を確認する
- 内容確定後、次のステップ（Notion書き戻し）の実施可否をユーザーに確認する

### 6. Notionへ書き戻し
ユーザー承認後、「日報自分用」ページの **先頭**（最新日報トグルの直前）に `N月M週目` トグルとして追記する。

- 週番号の算出: 対象期間の終了日（金曜日）が属する月で、その月の第何週目か（月初から数える）
  - 例: 2026-04-10（金）→ 4月2週目
- `mcp__claude_ai_Notion__notion-update-page` の `command: "update_content"` を使用
- `old_str` は既存の最新日付トグル開始行（例: `<details>\n<summary>4/9</summary>`）を指定
- `new_str` は「新トグル全文 + 元の `old_str`」を連結
- 週間レポートのトグル内では、各見出し行の先頭にタブ1つを付けてネスト配下として扱う

#### 書き戻しフォーマット例
```
<details>
<summary>4月2週目</summary>
	### 週間レポート（4/6〜4/10）
	**今週の主な取り組み**
	- ...

	**成果・ハイライト**
	- ...

	**反省点・課題**
	- ...

	**AI活用トピック**
	- ✅ ...
	- ⚠️ ...
</details>
```

## 注意事項
- 日報トグルが折りたたまれていても `notion-fetch` は中身を返すため、追加展開操作は不要
- 取得できた日付が対象期間より少ない場合は、必ずユーザーに明示する（「4/8〜4/9のぶんのみ取得できました」等）
- AI活用トピックは成功談/失敗談を ✅/⚠️ で区別するが、無理に両方揃えなくてよい
- 会社提出用フォーマット指定がある場合はユーザーに確認してから整形する
- Notion書き戻しは **ユーザーが内容にOKを出してから実施**。勝手に書き込まない
